# Strada.Brain Remediation Plan

> Snapshot note: This remediation plan tracks the 2026-03-17 stabilization wave. It is not the authoritative source for current runtime behavior or env defaults. For current behavior, use [README.md](../../README.md), [SECURITY.md](../../SECURITY.md), [src/config/README.md](../../src/config/README.md), [src/channels/README.md](../../src/channels/README.md), and [src/dashboard/README.md](../../src/dashboard/README.md).

## Working Rules

- No merge without mandatory review.
- Every phase must finish with targeted tests plus a human-readable verification note.
- Security and state-integrity fixes land before feature work.

## P0

- Close all fail-open approval/review paths.
- Remove auto-approve behavior from interactive channels.
- Restore broken encryption initialization and env helper behavior.
- Fix dashboard/setup paths that are currently unusable or misleading.
- Stop migrations and background execution from silently corrupting or losing state.

## P1

- Repair channel requester-binding across Slack, Discord, Telegram, WhatsApp, and Web.
- Fix deployment approval wiring and idempotency.
- Fix memory, learning, and RAG migration/data-loss paths.
- Align setup flows, validation boundaries, and configuration keys.
- Repair backend/frontend API contract mismatches and hidden-error UI behavior.

## P2

- Remove dead code and decorative infrastructure.
- Close documentation drift across README, SECURITY, CONTRIBUTING, and module READMEs.
- Expand coverage for portal, setup, memory, network, plugins, and shared infrastructure.
- Refactor oversized bootstrap/orchestrator surfaces after behavior is stable.

## Execution Order

1. P0 security and correctness fixes with tests.
2. P0 state-integrity and setup fixes with tests.
3. P1 channel and dashboard contract fixes.
4. P1 migration and persistence hardening.
5. P2 dead-code cleanup and architectural refactor.

## Exit Criteria

- No known fail-open approval or auth path.
- No known auto-approve timeout path.
- No known destructive migration path without preservation tests.
- Dashboard, setup, and primary channels verified by targeted integration coverage.
