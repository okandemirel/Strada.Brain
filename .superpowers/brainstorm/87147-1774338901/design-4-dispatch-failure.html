Failure Handling - 4 Seviye Kurtarma
1
Retry (aynı provider)
Geçici hata (timeout, rate limit, 5xx) → aynı provider'da 1x retry
maxRetries: 1 | backoff: 2s
2
Provider Escalation
Retry başarısız → ProviderAssigner'dan sonraki en iyi provider'ı al
Örnek: Groq fail → DeepSeek'e geç → Claude'a geç
Mevcut FallbackChainProvider mantığıyla uyumlu
3
Re-Decompose (mevcut GoalDecomposer reactive)
Provider escalation da başarısız → node'u daha küçük parçalara kır
GoalDecomposer.decomposeReactive() kullanır (mevcut, dokunulmaz)
Yeni child node'lara tekrar CapabilityMatcher + ProviderAssigner çalışır
maxRedecomposition: 2 per node
4
Criticality Check + Graceful Degrade
Re-decompose de başarısız → LLM ile criticality değerlendir
Non-critical: node skip, bağımlılar devam eder, kullanıcıya partial result
Critical: bağımlı node'lar da skip, kullanıcıya hata + tamamlanan kısımları sun
failureBudget: 3 (toplam max fail sayısı, mevcut GoalExecutor ile aynı)
Timeout, Cancellation & Budget
Per-Node Timeout
Her PAOR loop'a AbortController
Default: 120s (interactive), 300s (background)
Configurable per delegation type
Timeout → Level 2 escalation'a geç
Cancellation
Kullanıcı iptal ederse → tüm PAOR loop'lar abort
Tamamlanan node'ların sonuçları korunur
Partial result kullanıcıya sunulur
AbortController.abort() cascade
Budget Guard
Her node tamamlanınca maliyet hesapla
Toplam maliyet > agent budget → dur
Tamamlanan kısımları sun
Mevcut AgentBudgetTracker ile entegre
// Tam failure handling akışı
node execution fails:
  → L1 retry(sameProvider, maxRetries=1, backoff=2s)
  → L2 escalate(nextBestProvider from ProviderAssigner scores)
  → L3 redecompose(GoalDecomposer.reactive, maxRedecomp=2)
      → new children get CapabilityMatcher + ProviderAssigner
  → L4 criticalityCheck(LLM)
      → non-critical: skip node, dependents continue
      → critical: skip node + dependents, partial result
  → failureBudget exhausted: abort all, return completed work
